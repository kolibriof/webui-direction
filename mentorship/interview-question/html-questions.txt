4. What is the difference between a block-level element and an inline element? 
  - Block elements start in a new line and you can define a top and bottom margins for them. However, inline elements start next to the element and do not have a top and   bottom margins.

22. Why is it generally a good idea to position CSS <link>s between <head></head> and JS <script>s just before </body>? Do you know any exceptions?
  - It is a good idea because of the performance and page appearance. What may happen is that your CSS code may load after the HTML elements are loaded, which can result   in weird styling behavior. Generally, if you place script or styles after the body, the browser will render them in the last place. However, there a exceptions, one of   them applies to one page sites, where scripts and CSS styles being rendered on demand, rather than all at once.
